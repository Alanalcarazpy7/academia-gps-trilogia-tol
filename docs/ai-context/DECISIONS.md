# Decisiones confirmadas

## Arquitectura

- La solución será una sola landing page.
- No crear subrutas públicas para el contenido principal.
- Cada sección será un componente independiente y reutilizable.
- La landing debe poder convertirse más adelante en una página interna de Academia GPS sin rehacer todo.

## Pagopar

- Pagopar se usa mediante enlaces entregados por la clienta.
- No implementar API, webhook, carrito, checkout propio ni lógica de pagos.
- Cada botón de compra abre el enlace correspondiente.
- Los enlaces deben centralizarse en un archivo de configuración.

## WhatsApp

- Habrá botones para una o más asesoras.
- Cada botón debe abrir WhatsApp con un mensaje prearmado.
- Números y mensajes se centralizan en configuración.
- No inventar números, nombres ni enlaces pendientes.

## Contenido

- El Word de la clienta es la fuente principal para el copy.
- El PDF es fuente de apoyo para comprender el producto.
- No copiar páginas del PDF como imágenes para reemplazar contenido HTML.
- El contenido debe convertirse en secciones web legibles, persuasivas y fáciles de escanear.
- No inventar testimonios, credenciales, cifras o resultados.

## Precios y fechas del producto

- Mantenerlos editables desde un único archivo.
- Verificar con la clienta antes de publicar promociones, precios, cuotas, bonos y fechas.
- No publicar datos vencidos o no confirmados.

## Diseño

- Mantener identidad Academia GPS / Trilogía TOL.
- Evitar apariencia genérica de plantilla.
- Animaciones suaves y discretas.
- Priorizar velocidad, claridad y conversión.
